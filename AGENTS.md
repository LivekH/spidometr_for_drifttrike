## 1. Описание проекта

### Назначение

**Spidometr for Drifttrike** — приборная панель дрифттрайка на **Arduino Nano 3**.  
Один экран ILI9341 (**меню нет**): скорость, заряд АКБ (кулоновский счётчик), газ, TORQUE; плюс поворотники.  
В код — только через handoff; ниже — черновик идеи владельца (2026-07-19).

### Платформа и железо

| Компонент | Модель / детали | Пины (канон) |
|-----------|-----------------|--------------|
| MCU | Arduino Nano 3 (ATmega328) | — |
| Дисплей | ILI9341 (SPI TFT) + **Arduino_GFX** | SPI 11/12/13; DC **8**, CS **9**, RST **7** (провизорно, handoff `gui-static-mock`) |
| Датчик Холла (скорость) | SS49E | TBD (ADC) |
| Датчик тока | ACS712ELC-30A (±30 A), в разрыв батарея→контроллер | TBD (ADC) |
| Ручка газа | аналог (положение 0…100%) | TBD (ADC) |
| Напряжение АКБ | делитель / измерение (TBD) | TBD (ADC) |
| Концевик поворотник L | вход | **D2** (заявлено) |
| Концевик поворотник R | вход | **D3** (заявлено) |
| Реле поворотник L | выход меандр | **D9** (заявлено) |
| Реле поворотник R | выход меандр | **D8** (заявлено) |

> Пины дисплея/датчиков **TBD** — из Proteus или разводки, отдельным handoff.  
> D2/D3/D8/D9 — по идее владельца; подтвердить при фиксации карты пинов.  
> Пины и EEPROM **не менять молча**.

### Arduino_GFX: конфликт Nano ↔ Mega (важно)

Рабочая копия IDE (`Documents/Arduino/libraries/Arduino_GFX-master`) **пропатчена под Mega 2560 Pro**: в ветке `#else` (комментарий «Nano…») стоят пины Mega (`SCK52 MOSI51 MISO50 CS53 DC48 RST49`).  
Эталон со сток-пинами Nano: `Termoregular_V1.1/libraries for the project/Arduino_GFX-master/src/Arduino_GFX_Library.h` (внизу `#else`: 13/11/12/9/8/7/6).

- На **Termoregular (Mega)** `create_default_Arduino_GFX()` работает.
- На **Spidometr (Nano)** те же дефолты **не подойдут** (пинов 48…53 нет).
- Почему «автоопределение не вышло»: у stock GFX класс Mega часто попадает в тот же `#else`, что и Nano — отдельной ветки `__AVR_ATmega2560__` нет. Правка «заменить цифры в `#else`» ломает Nano.

**Выбранный путь для Spidometr (не ломаем Termoregular):**

- Рабочую библиотеку IDE **не откатываем** (остаётся патч Mega).
- В `sketch_jul19a` **не** вызывать `create_default_Arduino_GFX()`.
- Собирать шину / `Arduino_ILI9341` **явно** с пинами Nano (SPI 11/12/13 + CS/DC/RST из AGENTS, когда зафиксируем).

Альтернатива на будущее (отдельный handoff): `#elif` Mega + Nano в `#else` в одной библиотеке — оба проекта через `create_default`.

Пошаговая шпаргалка по `.h`: `Arduino_GFX_пины_платы.txt`.

### Память Nano (заметка)

У Termoregular на Nano flash упирался ~в 84% после меню времени/temp/hyst → переезд на Mega.  
Spidometr проще по UI, но **Arduino_GFX + ILI9341** сами по себе тяжёлые: следить за flash/SRAM в IDE, без лишних `String` и без раздувания меню без нужды.

### Контракт: роль Nano (черновик идеи)

1. **Сбор данных:** газ, напряжение АКБ, скорость (Холл), ток (ACS712).
2. **GUI:** скорость, уровень заряда, крутящий момент (TORQUE), положение газа; без меню.
3. **Поворотники:** входы D2/D3 → Arduino генерирует меандр на реле D8 (прав.) / D9 (лев.), заданная частота.

### Контракт: аналоговые входы и расчёты (черновик)

**Входы (ADC):**

| Величина | Откуда | Зачем |
|----------|--------|--------|
| Напряжение АКБ `U` | делитель → ADC | уровень, мгновенное падение U, мощность, SOC/прогноз |
| Ток `I` | ACS712 → ADC | потребление, кулон (А·ч), мощность, TORQUE |
| Газ `throttle` | ручка → ADC | шкала 0…100%, задание «оборотов» для TORQUE |
| Скорость `v` | SS49E (+ магнит/колесо) | спидометр, Вт·ч/км, запас хода |

**Производные (логика идеи владельца):**

- Потребление / мощность: из `I` и `U` (P ≈ I × U).
- Мгновенное падение напряжения: наблюдение просадки `U` под нагрузкой (вместе с `I`).
- Кулон / оставшаяя ёмкость: интегрирование `I` во времени → потраченные А·ч → вычитание из номинала → % заряда (стабильнее, чем только по `U`).
- Расход и запас хода: Вт·ч/км из мощности и `v`; км ≈ оставшиеся Вт·ч / средний расход.
- **TORQUE:** зависит от положения ручки газа («сколько оборотов задано») **и** фактической нагрузки (`I`, просадка `U`) — точная формула/единицы **TBD** (согласовать до handoff).
- Номинал ёмкости АКБ / сброс счётчика: без меню — способ **TBD** (константа / EEPROM позже).

## 3. Development Rules

### Перед любой задачей

1. Прочитать этот `AGENTS.md`.
2. Если задача меняет поведение / пины / EEPROM / UI — **сначала обновить AGENTS.md** (с владельцем).
3. Создать/обновить `handoffs/<task-slug>/handoff.md` (цель, acceptance, out of scope).
4. Писать код только после пунктов 1–3, против канона `sketch_jul19a/sketch_jul19a.ino`.

### Пошаговая работа (обязательно)

1. **Одна задача за раз.** Не паковать несколько фич в один заход.
2. Порядок: **спека (AGENTS + handoff) → код → заливка → проверка на железе → ок/правки → следующая задача.**
3. Следующую фичу не начинать, пока текущий handoff не закрыт (или владелец явно не переключил задачу).
4. Крупное дробить на маленькие проверяемые шаги.

### Жёсткие правила кода

1. Не менять пины / EEPROM без обновления `AGENTS.md`.
2. Не рефакторить «заодно». Один handoff = одна цель.
3. Память AVR: избегать лишних `String` в горячем пути; предпочитать `const char*`, `byte`.
4. Блокирующие `delay()` в `loop()` запрещены (кроме осознанного splash в `setup`, если описано в handoff).
5. Минимальный diff; без «заодно почищу».
6. Новые/изменённые строки кратко комментировать по функции; не дублировать одинаковые комментарии рядом.

### Правила для ИИ-агента

- Перед правкой: `AGENTS.md` + актуальный `handoff.md`.
- При неясности контракта — спросить, не угадывать.
- После изменения поведения — обновить AGENTS.md + статус handoff.
